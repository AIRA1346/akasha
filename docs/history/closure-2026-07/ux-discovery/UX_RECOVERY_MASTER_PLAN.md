## §1. 왜 AKASHA는 「지식 우주」처럼 느껴지지 않는가?

### 1.1 구조적 원인

AKASHA는 Constitution이 말하는 세 축—**기록·연결·발견**—을 코드에 이미 갖추고 있다. 그러나 UI는 이를 **동등한 여정**이 아니라 **관리 도구 모음**으로 배치한다.

```
헌법:     기록 ──→ 연결 ──→ 발견  (순환)
현재 UI:  발견 ──→ 관리 ──→ 편집  (단방향, 편집기로 수렴)
```

| 축 | 구현 | UI에서의 위치 | 체감 |
|----|------|---------------|------|
| 발견 | Fusion, catalog 10k, browse | 검색 다이얼로그, 휴리스틱 홈 섹션 | 「찾아서 열기」 |
| 연결 | wiki, link index, neighbors, graph | 편집기 하단, 사이드 메뉴, Beta 라벨 | 「숨은 기능」 |
| 기록 | Sanctum, vault, Records | 워크벤치 진입 후 | 「작업실」 |

**지식 우주**는 사용자가 한 노드에서 이웃으로 **걸어가며** 우주를 확장하는 경험이다. 현재는 노드를 **열어 폼을 채우는** 경험에 가깝다.

### 1.2 경험적 원인 (5가지)

1. **이중 네비게이션** — 사이드바(서재·컬렉션·그래프·타임라인)와 하단 탭(홈·탐색·검색·라이브러리·컬렉션)이 같은 공간을 다른 이름으로 가리킨다. mental model이 「우주」가 아니라 「앱 메뉴」다.

2. **클릭 정책 비대칭** — Home Work는 프리뷰(탐험), Entity·검색은 워크벤치(편집) 직행. 「탐험 → 깊이」 리듬이 화면마다 다르다.

3. **카피–구현 불일치** — 「발견의 여정」은 추천이 아닌 정렬 휴리스틱, 「그래프」는 리스트, 「Ctrl K」는 미연결. 신뢰가 깨지면 우주 메타포도 공허해진다.

4. **연결의 매몰** — `[[wiki]]`, incoming links, link neighbors는 **이미 동작**하지만 마크다운 툴바·Info 패널 하단·사이드바 깊숙이에 있다. 사용자는 「연결할 방법」을 모른다.

5. **관리 도구의 동등 노출** — 볼트 설정, 서재, 커스텀 대시보드, 동기화·AI 가져오기가 탐험과 같은 시각적 무게를 갖는다. Netflix가 아니라 Notion에 가깝다.

### 1.3 문서–코드 갭의 부수 효과

`CURRENT_STATE.md` / `ROADMAP.md`는 Phase 3·4·5를 「미착수」로 표기하지만, Person·wiki·Records·link index는 **이미 동작**한다. 팀·사용자 모두 「아직 없는 기능」으로 인지해 **있는 연결 UX를 홍보하지 못한다**.

## §2. 현재 기능 중 무엇이 숨겨져 있는가?

### 2.1 기능 가시성 매트릭스

| 기능 (구현됨) | 진입점 | 가시성 | 숨김 이유 |
|---------------|--------|--------|-----------|
| `[[wiki]]` 엔티티 링크 | Sanctum 툴바·슬래시 | **낮음** | 온보딩·CTA 없음 |
| Entity link picker | 마크다운 편집 중 | **낮음** | 워크벤치 진입 필요 |
| Incoming links | Info 패널 하단 | **낮음** | 메타 폼 아래 스크롤 |
| Link neighbors (인물·연결 작품) | 프리뷰·워크벤치 Info | **중간** | 링크 없으면 빈 상태; 홈 Work만 |
| Knowledge Graph (연결 맵) | 사이드바·빠른 액션 | **중간** | Beta 라벨, 하단 탭 없음, 그래프 기대 |
| Records / Timeline | 사이드바·빠른 액션 | **중간** | 하단 탭 미표시 |
| Fusion Search (10k+catalog) | TopBar·FAB·빠른 액션 | **높음** | — |
| Collection filter (`relatedWorkId`) | 컬렉션 설정 | **낮음** | Cast 큐레이션 맥락 필요 |
| Recent exploration + 진행률 | 홈·사이드바 | **높음** | 진행률 의미 모호 |
| TodayRecallCard | — | **없음** | `showRecallCard=false` |
| AppBar 도구 (sync, AI import, inbox) | AppBar | **없음 (홈)** | 홈 AppBar null |
| Ctrl+K 전역 검색 | TopBar 배지 | **거짓** | Tab만 바인딩 |
| Dashboard Preview Panel | 홈 Work 탭 | **중간** | 홈 전용; 다른 browse 없음 |
| Same-day records | Info 패널 | **낮음** | 라벨이 약함 |
| Wiki 링크 탭 (읽기) | Sanctum 보기 모드 | **중간** | 편집 모드와 혼재 |

### 2.2 「숨김」의 공통 패턴

1. **워크벤치 안에만 존재** — 탐험은 홈, 연결은 편집기. 축이 공간적으로 분리됨.
2. **빈 상태 = 끝** — 링크 없을 때 「다음에 할 일」이 없음.
3. **Beta / 미구현 카피** — 실제로는 동작하는 기능을 불완전하다고 스스로 라벨링.
4. **문서가 없다고 말함** — wiki·Entity·Timeline이 로드맵에서 미착수.

## §3. 어떤 화면이 사용자를 탐험으로 유도하지 못하는가?

| 화면 | 탐험 실패 이유 | 개선 방향 (UX만) |
|------|----------------|------------------|
| **Home Dashboard** | 통계·휴리스틱 추천·이중 네비가 「상태 점검」 느낌 | 탐험 히어로(최근+프리뷰) 중심 IA |
| **Search (Fusion)** | 결과 클릭 → 워크벤치 직행, 연결 맥락 없음 | 선택적 peek 또는 「연결 N건」 힌트 |
| **Explore Grid** | 프리뷰 없음, Work/Entity 정책 불일치 | 홈과 동일 peek 정책 |
| **Work Workbench** | Info 폼이 상단 60%, 연결은 하단 | 연결 섹션 상단 승격 |
| **Entity Workbench** | Work 대비 이웃 없음, 타입 무차별 폼 | incoming 강조 + 타입별 카피 |
| **Collection** | 큐레이션 도구 UI, 탐험 스토리 없음 | 유지 (분리 축) — 홈에서 중복 노출 금지 |
| **Knowledge Graph** | 그래프 기대 vs 리스트, 빈 연결 시 막다른 길 | 「연결 맵」카피, 빈 상태 CTA |
| **Records** | 사이드바 깊숙, Phase 미착수 인식 | 「기록」축 홈 빠른 액션과 정합 |
| **Library / Vault** | 서재 관리, 메타 편집 | 탐험 유도 대상 아님 — 네비 정리 |

**가장 큰 단절 3곳**

1. **홈 프리뷰 ↔ 워크벤치** — 유일한 탐험 중간층이 홈에만 있음.
2. **연결 UI ↔ 빈 볼트** — 링크 없는 사용자는 연결 기능 존재를 알 수 없음.
3. **발견의 여정 ↔ 실제 추천** — 기대와 다른 콘텐츠로 발견 축 신뢰 하락.

```
└──────────────────────────────────────────────────────────────────────────┘
```

**진입:** 홈 빠른 길·사이드바 단일 경로 (중복 제거). Beta 라벨 제거.

## §5. 새 기능 없이 UX만 바꾸면 체감이 얼마나 개선되는가?

### 5.1 정량적 추정 (휴리스틱)

| 영역 | 현재 체감 (1–10) | UX-only 후 | 근거 |
|------|------------------|------------|------|
| 「우주」메타포 | 3 | **6–7** | 궤도·연결 맵·프리뷰 일관화 |
| 연결 기능 발견 | 2 | **7** | 상단 승격·빈 상태 CTA·카피 정직화 |
| 발견 경험 | 6 | **7** | Fusion 유지; 홈 휴리스틱 정리 |
| 기록 경험 | 7 | **8** | Sanctum 명명·탭 breadcrumb |
| 전체 NPS 예상 | — | **+25~40%** | 숨은 기능 가시화가 핵심 |

### 5.2 개선되지 **않는** 것 (명시)

- 진짜 추천·유사도 엔진
- 3D·force-directed 그래프
- 링크 없는 콜드스타트 볼트의 **풍부한** 연결 환상
- Mixed Library, 신규 엔티티 타입

### 5.3 ROI가 높은 UX 변경 Top 7 (구현 난이도 낮음 → 높음)

| # | 변경 | 효과 | 코드 규모 추정 |
|---|------|------|----------------|
| 1 | 카피 정직화 (그래프→연결 맵, 발견의 여정 축소) | 신뢰 회복 | 문자열 |
| 2 | 워크벤치 Info 패널 **순서** 재배치 | 연결 가시화 | 레이아웃 |
| 3 | 빈 연결 상태 CTA | 다음 행동 | 위젯 카피 |
| 4 | 홈 IA 통합 (발견 3탭 제거) | 인지 부하 ↓ | 홈 위젯 |
| 5 | 프리뷰 정책 Explore·Search 확장 | 탐험 일관 | 네비게이션 |
| 6 | Ctrl+K 바인딩 또는 배지 제거 | 약속 이행 | 단축키 |
| 7 | 이중 네비 정리 | mental model | IA 리팩터 |

**결론:** 기능 추가 없이도 **체감 가치의 60~70%**를 회복할 수 있다. 나머지 30%는 링크 밀도·추천 품질 등 **콘텐츠·알고리즘** 영역이다.

## §6. AKASHA v2 UX 구조

### 6.1 공간 모델 (3층)

```
┌─────────────────────────────────────────┐
│  L1  탐험층   Home · Explore · Search   │  → peek · 연결 힌트 · 발견
├─────────────────────────────────────────┤
│  L2  중간층   Preview Panel (공통)       │  → 연결 따라가기 · 상세 결정
├─────────────────────────────────────────┤
│  L3  성역층   Workbench · Records        │  → 기록 · 링크 생성 · 깊이
└─────────────────────────────────────────┘
```

**큐레이션(Collection)·관리(Library/Vault)** 는 L1 사이드바 — 탐험 축과 **평행**, 홈 본문에 섞지 않음.

### 6.2 네비게이션 원칙

| 레이어 | 역할 | 항목 |
|--------|------|------|
| **하단 탭** | 공간 전환 | 홈 · 탐색 · 검색 · 라이브러리 · 컬렉션 |
| **사이드바** | 맥락·히스토리 | 최근 탐험 · 서재 · 컬렉션 · (연결 맵·기록 링크) |
| **탭 레일** | 탐험 스택 | breadcrumb 스타일 경로 |

### 6.3 헌법 순환 UI

```
발견 (검색·탐색·오늘의 연결)
    ↓ peek
연결 (프리뷰 이웃 · 연결 맵 · incoming)
    ↓ 기록하기
기록 (Sanctum · Records)
    ↓ [[wiki]]
연결 (다시 L2)
```

### 6.4 화면별 한 줄 역할

| 화면 | v2 역할 |
|------|---------|
| Home | 지식 우주 **진입점** — 최근·연결·우주·빠른 길 |
| Preview | **연결 따라가기** 허브 |
| Workbench | **기록** SSOT + 연결 생성 |
| Entity View | **노드** — incoming/outgoing 관계 |
| 연결 맵 | 볼트 전체 **연결 조망** (리스트) |
| Collection | **큐레이션** (탐험과 분리) |
| Records | **시간축 기록** |

### 6.5 Sprint 이후 권장 작업 순서 (구현 Phase — 본 Sprint 범위外)

| 우선순위 | 항목 | 유형 |
