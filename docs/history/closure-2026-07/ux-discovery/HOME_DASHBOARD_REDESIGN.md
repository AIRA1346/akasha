# Home Dashboard UX Audit & Redesign

> **Sprint:** UX Recovery — Step 2  
> **갱신:** 2026-06-22  
> **전제:** 새 기능 없음. 정보 구조·카피·배치·흐름 재설계만.

---

## 현재 홈의 정체성 문제

홈은 시안상 「지식 우주 진입점」이지만, 실제 정보 구조는:

```
환영
+ 최근/탐색 (개인 히스토리)
+ 발견 (휴리스틱)
+ 통계 (볼트 카운트)
+ 빠른 액션 (기능 바로가기)
+ 사이드바 (서재·컬렉션·대시보드 관리)
```

→ **Netflix형 탐험**과 **Notion형 관리**가 한 화면에 경쟁한다.

**헌법 질문:** 홈에서 사용자의 첫 질문은 무엇인가?

| 현재 암시적 답 | 이상적 답 |
|----------------|-----------|
| 「내 라이브러리 상태가 어떠니?」 | 「오늘 무엇을 탐험할까?」 |
| 「어디로 갈까?」(5개 액션) | 「무엇이 나와 연결되어 있니?」 |

## 섹션별 Audit

### 1. 계속 탐험하기

| 항목 | 내용 |
|------|------|
| **데이터** | `RecentExplorationStore` (열람 기록), 진행률=기록 풍부도 휴리스틱 |
| **행동** | Work 탭 → 프리뷰; 「더 보기」→ 탐색 그리드 |
| **비전** | 발견+기록 △ — 「탐험」 메타포는 맞으나 진행률 의미 모호 |

**판정: 수정**

- 유지: 최근 열람 기반 카드, 프리뷰 연동
- 수정: 섹션명을 「최근 탐험」으로 정직화 **또는** 진행률을 「기록 깊이」로 라벨 변경
- 수정: 빈 상태 시 「작품을 열어 탐험을 시작하세요」+ **검색 CTA** (탐색 그리드 대신)
- 제거: `addedAt` 정렬과 혼동될 수 있는 「계속」이라는 단어 (이미 store로 통합됨 — OK)

---

### 2. 사이드바 「최근 탐색」

| 항목 | 내용 |
|------|------|
| **데이터** | 동일 `RecentExplorationStore` |
| **행동** | 탭 → `openRecentExploreItem` |
| **비전** | 발견 △ |

**판정: 수정**

- 유지: 데이터 소스 (홈 카드와 SSOT 일치 — 최근 개선됨)
- 수정: 홈 「계속 탐험하기」와 **중복** — IA상 하나로 통합 검토
  - 옵션 A: 사이드바는 축약(2건), 홈은 히어로(4건)
  - 옵션 B: 사이드바만 유지, 홈 섹션 제거 → 화면 여백을 「연결 맵」에 할애
- 수정: 썸네일 있음 — 좋음; 홈 카드와 시각 일관성 유지

---

### 3. 주요 인물 / 연결 작품 (프리뷰 패널)

| 항목 | 내용 |
|------|------|
| **위치** | `DashboardPreviewPanel` (홈 Work 선택 시만) |
| **데이터** | `link_index` + 태그 fallback |
| **비전** | 연결 ✅ (링크 있을 때) |

**판정: 수정 (홈 본문이 아닌 프리뷰 소속 — 유지)**

- 유지: 프리뷰 내 이웃 섹션 — **연결 가치의 홈 노출점**
- 수정: 링크 없을 때 「이 작품에 인물 연결하기」→ 워크벤치 본문 편집 유도 (카피만)
- 제거: 홈 대시보드 본문에 별도 「주요 인물」 섹션 추가 **금지** (중복)

---

### 4. 컬렉션 (사이드바 + 하단 탭)

| 항목 | 내용 |
|------|------|
| **홈 본문** | 컬렉션 섹션 **없음** |
| **접근** | 사이드바 「내 컬렉션」, 하단 탭 |
| **비전** | 큐레이션 — 홈 탐험과 다른 축 |

**판정: 수정 (홈에서 분리 유지)**

- 유지: 홈 본문에 컬렉션 그리드 넣지 않음
- 수정: 홈 「빠른 액션」에 컬렉션 없음 — 하단 탭으로 충분; 사이드바 명칭을 헌법 용어에 맞게 정리
- 수정: 빈 컬렉션 → 생성 다이얼로그 (구현됨) — 홈에서 「내 큐레이션」1줄 요약 링크만 고려

---

### 5. 발견의 여정

| 항목 | 내용 |
|------|------|
| **탭 0 추천 연결** | 최근 추가 3쌍 + 태그/창작자 문자열 |
| **탭 1 새로운 작품** | `addedAt` 상위 3 |
| **탭 2 주목할 인물** | 카탈로그 person 상위 3 |
| **비전** | 발견 ✗ (추천 엔진 아님) |

**판정: 수정 (또는 대폭 축소)**

- 수정: 「추천 연결」→ 「볼트 안의 연결」 (링크 인덱스 기반 페어만 표시) — **로직 변경 없이** 빈/약한 결과 시 섹션 숨김
- 수정: 유사도 % 카피 제거 (현재 없음 — 유지)
- 제거 검토: 탭 1「새로운 작품」은 「최근 추가」섹션과 **중복** → 통합
- 제거 검토: 탭 2는 「인물 탐색」빠른 액션과 중복

**대안 IA:** 발견의 여정 → **「오늘의 연결」** 단일 행 (링크 1~3건만, 없으면 숨김)

### 6. 지식 우주 현황 (통계)

| 항목 | 내용 |
|------|------|
| **궤도 위젯** | vault+catalog 엔티티 타입별 카운트 |
| **최근 추가** | `addedAt` 5건 + NEW 뱃지 |
| **비전** | 발견 △ (규모 감) / 기록 △ |

**판정: 수정**

- 유지: 궤도 — 「우주」 메타포의 시각적 앵커 (유일)
- 수정: 숫자를 **탐험 CTA**와 연결 (예: 「인물 12 → 인물 탐색」)
- 수정: 「최근 추가」를 발견의 여정과 **통합**하여 한 곳만 유지
- 제거 검토: 시안식 대규모 가짜 통계 — **이미 실데이터** (유지)

### 9. 하단 탭 vs 사이드바

| 항목 | 하단 | 사이드바 |
|------|------|----------|
| 홈 | ✅ | ✅ |

## 「관리 대시보드」→「지식 우주 진입점」 전환 가능성

### 가능함 — 조건

1. 홈 본문에서 **관리 목록**(서재·컬렉션·커스텀 대시보드) 비중 축소 → 사이드바 전용
2. **발견의 여정** 정직화 또는 축소 — 거짓 추천 제거
3. **프리뷰 패널**을 홈의 중심 상호작용으로 격상 (이미 Work에 적용됨)
4. **연결**이 비어 있을 때의 빈 상태를 「다음 탐험 단계」로 디자인 (링크 넣기 유도)
5. 통계(우주) → 탐색 CTA로 연결

### 불가능한 것 (기능 추가 없이)

- 진짜 추천 엔진
- 3D 그래프
- 링크 없는 사용자에게 풍부한 「연결」 환상

## 제안 홈 IA (v2 방향)

```
[검색 — 전폭 CTA]

[최근 탐험 — 4카드 + 프리뷰]

[오늘의 연결 — 링크 기반 0~3 (있을 때만)]

[지식 우주 — 궤도 + 타입별 탐색 링크]

[빠른 길 — 검색 | 인물 | 연결 맵 | 기록(타임라인)]
```

**제거·통합 후보**

- 발견의 여정 3탭 → 「오늘의 연결」+ 최근 추가 통합
- 사이드바 최근 탐색 vs 홈 카드 중복 → 역할 분리
- 홈 AppBar 부재 → 최소 「⋯」메뉴로 동기화·가져오기
